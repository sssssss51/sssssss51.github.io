레이아웃 정의 공식 가이드

https://developer.apple.com/design/human-interface-guidelines/components


[Part01] Boxes

- 상자 사이즈 작게, padding 적극 이용
- 기본적으로 IOS와 IPadOS는 삼차 배경색 사용 (눈에 덜 띄는 색으로 배경 지정)


[Part02] Collections

- 표준 행, 그리드 레이아웃 사용 (간단한 모양) 패딩 적극 활용
- 텍스트 -> 테이블 / 이미지 -> 컬렉션
- 삽입, 삭제, 재정렬에 대한 애니메이션 추가 - 동적 레이아웃 변경 주의


[Part03] Disclosure controls

- 세부 정보는 숨기고 주요 내용에 공개 제어 기능 사용
- swiftUI에서 제공하는 보기 기능

https://developer.apple.com/documentation/SwiftUI/DisclosureGroup


[Part04] Labels

- 개발자 참고 사항이 있음 (swiftUI : Label, Text)
- 기본적인 시스템 글꼴 선호 (Dynamic Type)

소량 대량
편집 X label text view
편집 O text field text view



[Part05] Lists and Tables

- 그룹화 스타일에선 header, footer 추가
- 행의 추가적인 내용 표시 경우에만 info button 사용 가능

https://developer.apple.com/documentation/uikit/uitableviewcell/accessorytype/disclosureindicator


[Part06] Split views

- 사이드바 인터페이스 만들 때 (아니어도 사용하는 것을 권장)
